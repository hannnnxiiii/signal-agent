# GitHub Trending 日报 - 2026-03-14

## 概览
今日 GitHub Trending 呈现 AI Agent 基础设施爆发态势。从浏览器自动化到 RAG 平台，从 Prompt 测试到自然语言 UI 控制，开发者正围绕 Agent 的核心能力——感知、推理、执行——构建完整工具链。值得注意的是，阿里巴巴的 page-agent 与 Google 的 A2UI 同时上榜，标志着「自然语言驱动界面交互」正成为前端领域的新前沿。

## 精选仓库
### 1. [langflow-ai/openrag](https://github.com/langflow-ai/openrag)
- 分类：agent
- 简介：基于 Langflow、Docling 和 Opensearch 的一站式 RAG 平台
- 值得关注：RAG 是当下 Agent 落地的核心模式，但技术栈碎片化严重。openrag 将文档解析、向量检索、流程编排打包为单一方案，降低了企业级知识库 Agent 的构建门槛。

### 2. [lightpanda-io/browser](https://github.com/lightpanda-io/browser)
- 分类：agent
- 简介：专为 AI 与自动化设计的 headless 浏览器
- 值得关注：传统浏览器为人工浏览优化，对 AI Agent 而言过于笨重。Lightpanda 用 Zig 重写，CDP 协议兼容 Puppeteer/Playwright，意味着 Agent 可以更低成本地「看见」和操作网页，是浏览器自动化的新一代基础设施。

### 3. [promptfoo/promptfoo](https://github.com/promptfoo/promptfoo)
- 分类：agent
- 简介：面向 Prompt、Agent 和 RAG 的测试与红队评估框架
- 值得关注：Agent 能力越强，安全风险越高。promptfoo 提供声明式配置和 CI/CD 集成，支持多模型对比、对抗性测试和漏洞扫描。它填补了 AI 应用从「能跑」到「可生产」之间的关键空白。

### 4. [google/A2UI](https://github.com/google/A2UI)
- 分类：frontend
- 简介：Google 开源的 AI-to-UI 生成方案
- 值得关注：尽管仓库描述为空，但 A2UI 的命名和 13k+ star 表明这是 Google 在「自然语言生成界面」方向的探索。对于前端开发者而言，这代表了 UI 开发范式的潜在转变——从手写代码到 AI 生成可交互组件。

### 5. [alibaba/page-agent](https://github.com/alibaba/page-agent)
- 分类：agent
- 简介：JavaScript 页面内 GUI Agent，用自然语言控制网页界面
- 值得关注：与需要浏览器扩展或外部驱动的方案不同，page-agent 作为 JS 库直接运行在页面内，让任何网页都能被自然语言操控。这对自动化测试、无障碍访问、以及「AI 助手操作网页」场景极具想象空间。

## 结语
Agent 不再只是聊天机器人，它正在演变为能浏览网页、操作界面、检索知识的数字助手。今天的 Trending 列表揭示了这一趋势背后的工具链正在快速成熟——从底层浏览器到上层测试框架，从 RAG 基础设施到前端交互革新。对于开发者而言，是时候将 Agent 能力纳入产品架构的考量了。
